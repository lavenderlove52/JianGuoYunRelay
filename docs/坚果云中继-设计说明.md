## 1. 文档目的

本文档汇总 **WebDAV 中继（方案 B）**、**与坚果云对接方式**、**推送前版本校验（不一致则失败、不写远端）** 等约定，供实现与部署前评审。评审通过后作为开发与联调的依据。

---

## 2. 背景与目标

### 2.1 背景

- 公司网络 **无法直接访问** 坚果云 WebDAV（`dav.jianguoyun.com` 等）。
- 需在 **可访问的云服务器** 上部署中继：公司端通过 **WebDAV 连接中继**，由中继 **代为访问** 坚果云上的 KeePass 数据库文件（`.kdbx`）。
- 使用者为 **单人**；不设计高并发，但仍需避免 **在不知情情况下覆盖其他端已写入的远端文件**。

### 2.2 目标

1. 公司端 KeePass（或系统 WebDAV 客户端）将 **WebDAV 根地址配置为中继**，即可读写网盘上的 **固定路径** 数据库文件（经中继转发）。
2. 向坚果云 **推送（PUT）** 前进行 **版本核对**；若与本地编辑所依据的版本 **不一致**，则 **拒绝本次写入**（不向坚果云发送覆盖性 PUT），并返回明确错误码，由客户端重试拉取与人工/合并流程处理。
3. 通信 **加密**；中继访问坚果云使用 **应用密码**；不在日志中泄露密钥与文件内容。

### 2.3 非目标（本期不包含）

- 多用户租户隔离、配额计费、审计报表。
- 自动合并两端 `.kdbx` 条目级冲突（KeePass 合并能力由客户端操作，不由本协议自动完成）。
- 完整通用 WebDAV 服务器（仅实现 **KeePass / 常见客户端访问该文件所需的最小子集**，见第 6 节）。

## 3. 总体架构

```
┌─────────────┐     WebDAV(HTTPS)      ┌──────────────────┐     WebDAV(HTTPS)     ┌─────────────┐
│  公司电脑    │ ────────────────────► │  Rust 中继服务    │ ───────────────────► │   坚果云     │
│ KeePass 等   │ ◄──────────────────── │ （本设计范围）    │ ◄──────────────────── │ 固定路径kdbx │
└─────────────┘                        └──────────────────┘                        └─────────────┘
```

- 公司端 **不直连** 坚果云；仅与中继建立 TLS 连接。
- 中继使用配置的 **坚果云账号 + 应用密码**，访问配置中的 **远程绝对路径**（如 `/yourfolder/db.kdbx`）。

---

## 4. 数据流说明

| 阶段 | 行为 | 说明 |
|------|------|------|
| 打开库 | 客户端对中继发起 **GET**（或客户端内部等价流程） | 中继 **GET** 坚果云同一路径，将文件体与 **ETag**（若上游提供）返回给客户端。 |
| 保存库 | 客户端对中继发起 **PUT**，请求体为完整 `.kdbx` | 中继先按第 7 节做 **版本校验**，通过后再 **PUT** 至坚果云；失败则不写坚果云。 |

KeePass 具体触发「打开前拉取 / 保存后上传」取决于客户端与插件行为；本服务保证：**对中继的 WebDAV 语义与第 6、7 节一致**。

## 5. 安全设计

### 5.1 传输加密（HTTPS）

- 公司端与中继之间 **必须为 HTTPS**（TLS）。
- **推荐**：公网入口使用 **Nginx / Caddy** 终止 TLS，Rust 进程监听内网或本机端口。
- **可选**：由 Rust 进程直接绑定 443 并加载证书（运维复杂度略高，评审时二选一固化）。

### 5.2 中继访问坚果云

- 使用坚果云 **应用密码**，仅存于服务器环境变量或受控密钥管理，**不入库、不写日志**。
- 对上游请求使用 **TLS**；建议为上游 `reqwest` 等客户端配置 **合理超时**，避免长时间挂死。

### 5.3 公司端访问中继的鉴权

在 WebDAV 之上增加 **一层仅你使用的凭据**（与坚果云账号分离），例如：

- **方案 S1（推荐）**：WebDAV 的 **Basic Auth** 或 **Bearer**（若客户端支持自定义头则可用；KeePass 常见为 URL 内嵌用户口令或 Basic）。
- 凭据为 **高强度随机口令**，仅存环境变量；中继校验失败返回 **401**。

（评审需确认：KeePass 2 使用的插件/内置 WebDAV 是否支持除 URL 用户口令外的方案；若不支持，则采用 URL 路径分段密钥等需另行评估风险。）

### 5.4 日志与隐私

- 记录：时间、方法、路径模板、状态码、耗时、可选 `ETag` 存在性（**不记录 ETag 明文值**可配置，若需排障可短期打开脱敏策略）。
- **禁止**：请求/响应 body、坚果云密码、中继鉴权口令全文。

### 5.5 其他

- **请求体大小上限**（如 32MB），防止异常大包占用内存。
- 可选：**单 IP 简单限流**，降低扫描与误用风险。

## 6. WebDAV 子集与路径映射

### 6.1 路径映射（评审待定其一）

- **M1**：中继对外路径与坚果云路径 **1:1**（例如中继 `https://relay/dav/yourfolder/db.kdbx` 映射到坚果云 `/yourfolder/db.kdbx`）。  
- **M2**：中继 **固定单文件 URL**（如 `https://relay/vault.kdbx`），内部写死坚果云远程路径。  

推荐 **M2** 降低配置错误面；若需多文件再扩展 M1。

### 6.2 建议实现的 HTTP/WebDAV 方法

以 KeePass 通过 WebDAV 访问单文件常见需求为基线（**以联调实测为准增删**）：

| 方法 | 用途 | 中继行为概要 |
|------|------|----------------|
| **OPTIONS** | 客户端探测 | 返回允许的方法与 `DAV` 头（若需要）。 |
| **HEAD** | 取元数据（长度、ETag） | 转发至坚果云 HEAD；头字段按需透传。 |
| **GET** | 下载整文件 | 转发 GET；**透传 `ETag`**（若有）。 |
| **PUT** | 上传整文件 | 见第 7 节；通过后转发 PUT。 |
| **PROPFIND** | 部分客户端列举/探测 | 可返回 **单文件** 最小 XML 或 **深度 0** 响应；**需与 KeePass 实测对齐**。 |

不承诺完整 CalDAV/锁定扩展；若客户端强制要求 **LOCK/UNLOCK**，列入 **待验证** 并在评审中决定是否纳入一期。

## 7. 版本与防覆盖策略（核心）

### 7.1 原则

- `.kdbx` 为 **整文件替换**；无版本校验时 **后写覆盖先写**。
- 本期要求：**推送前若判定远端版本与「本次编辑所基于的版本」不一致，则本次推送失败，且不对坚果云执行覆盖写。**

### 7.2 版本标识

- **首选**：HTTP **`ETag`**（坚果云 WebDAV 响应头；以实测为准）。  
- **备选**：若某操作无 `ETag`，可用 **`Last-Modified`** 作弱校验（易有秒级竞态，仅作补充）。

### 7.3 校验方式（评审推荐组合）

**层 A — 标准条件写（优先）**

- 客户端在 **GET** 后保存 `ETag`；**PUT** 时携带 **`If-Match: "<etag>"`**。  
- 中继将 **`If-Match` 原样转发** 至坚果云 `PUT`。  
- 若坚果云判定前提失败，应返回 **412 Precondition Failed**；中继将该状态 **原样或映射** 返回客户端，且 **未发生覆盖写**。

**层 B — 中继强制预检（推荐与 A 同时启用）**

在转发 `PUT` 至坚果云 **之前**：

1. 对同一远程路径发起 **HEAD**（或轻量 GET）获取 **当前 `ETag`**。  
2. 与请求中声明的 **基准版本** 比较：  
   - 基准来源可为 **`If-Match` 头**；或客户端自定义头 **`X-Base-ETag`**（当客户端无法发 `If-Match` 时的退路，需评审是否接受）。  
3. 若 **不一致**：立即返回 **409 Conflict** 或 **412 Precondition Failed**（文档中 **固定一种** 便于客户端脚本处理），**不向坚果云发送 PUT**。  
4. 若 **一致**：再执行带 **`If-Match`** 的 PUT（若上游支持），或执行普通 PUT（行为以坚果云实测为准）。

**层 B 的目的**：即使客户端未正确转发条件头，只要约定携带「下载时的 ETag」，中继仍可拒绝危险覆盖。

### 7.4 失败后的用户侧动作（非服务实现，但需写入用户说明）

1. 重新 **GET** 最新文件。  
2. 在 KeePass 中使用 **比较/合并** 或人工处理。  
3. 再次保存并携带 **新 ETag** 重试 PUT。

## 8. 配置项一览（建议环境变量）

| 变量 | 含义 | 示例/备注 |
|------|------|-----------|
| `LISTEN_ADDR` | 中继监听地址 | `127.0.0.1:8080`（前置 TLS 代理时） |
| `JGY_WEBDAV_ROOT` | 坚果云 WebDAV 根 URL | `https://dav.jianguoyun.com/dav/` |
| `JGY_USERNAME` | 坚果云账号（邮箱） | |
| `JGY_APP_PASSWORD` | 应用密码 | 仅服务器侧 |
| `JGY_REMOTE_PATH` | 网盘内文件路径 | `/folder/db.kdbx` |
| `RELAY_AUTH_USER` / `RELAY_AUTH_PASSWORD` | 公司端访问中继的凭据 | 或使用单一 `RELAY_BEARER_TOKEN`（视 WebDAV 客户端能力） |
| `MAX_BODY_BYTES` | PUT 最大字节 | 如 `33554432` |
| `UPSTREAM_*_TIMEOUT` | 连接/读超时 | 秒级 |

（命名在实现阶段可微调，评审确认语义即可。）

## 9. 部署与运维

- 进程管理：systemd / 容器均可；需 **自动重启** 与 **日志轮转**。  
- 证书：Let's Encrypt 等，在 TLS 终止层续期。  
- 监控：进程存活、错误率、412/409 比例（异常升高可能表示多端同时改库）。  
- 备份：坚果云侧文件版本由网盘产品提供；中继 **不替代** 网盘历史版本策略。

---

## 10. 风险、依赖与待验证项

| 项 | 说明 |
|----|------|
| 坚果云 WebDAV 语义 | **`If-Match` on PUT**、**HEAD 是否稳定返回 ETag** 必须 **用真实账号与路径实测**；若不完整，仅依赖层 B 的 HEAD 比对。 |
| KeePass 行为 | 是否发送 `If-Match`、是否依赖 **PROPFIND/LOCK**；决定一期必须实现的方法集合。 |
| 整文件模型 | 版本校验只能 **拒绝覆盖**，不能自动合并业务数据。 |
| 单点故障 | 中继或云主机不可用则公司端无法同步；需接受或准备备用通道。 |

## 11. 实现模块划分（供开发对照）

| 模块 | 职责 |
|------|------|
| `config` | 加载第 8 节配置，启动校验 |
| `auth` | 公司端访问中继鉴权 |
| `upstream` / `jgy` | 对坚果云发起 GET/HEAD/PUT，处理超时与错误映射 |
| `webdav` | 对外 WebDAV 子集：路由、XML 构造（PROPFIND 等） |
| `version_guard` | 第 7.3 节层 B：PUT 前 HEAD 比对 + 与 If-Match 协同 |
| `main` | TLS/监听（若自管）、合并路由、tracing |
